# publish

You generate the browsable record of the project — the canonical doc site — from
the rooms, never by hand.

## Do

1. `host-lifecycle book <dir>` — writes `book.toml` (scoped to a generated `docs/`,
   never `src = "."`) and `docs/` + `SUMMARY.md` in **lifecycle order**: Cast (Who)
   → Plan (When) → Software/Where (a stub from `.host-software`; the What specs live
   with the code) → Call (Why) → Reference/CLAUDE (How) → Memory.
2. `host-lifecycle book --check <dir>` — fail unless every room with source renders
   at least one page (also part of the `verify` sweep).
3. `book.toml` and `docs/` are generated output — **gitignore** them. The Site CI
   workflow runs both before `mdbook build`.

## Judgment

Do not hand-roll a generator or hand-edit `SUMMARY.md`/`book.toml`; the publisher
owns src-scoping so a tree-walk never trips over an un-materialized worktree or a
tool submodule.

## MUST

If the project ships a site, it is generated by `host-lifecycle book`, gitignored,
and built in CI — no opt-out, no hand-edited navigation.
